---
开发完成,进入交付待审:

1. `git status`,并用 `git diff main...HEAD` 展示本分支的完整改动。
2. 运行测试 / lint / 构建,报告结果;有失败先修再继续。
3. 用一段话摘要:做了什么、为什么这样做、影响面、我需要重点看的地方。
4. 确认当前在功能分支上、改动已提交到该分支。

然后**停下**。不要 push、不要 merge 到 main。

告诉我:确认无误后,由我本人执行推送 / 合并(我会用 `approve-push` 或自己 push)。
